# Learning Profile — Casey

Concise learning context for explanations and tutoring. Current builders inject
all `memory/*.md`, so keep this short.

## Baseline
- Capable full-stack developer; strongest in practical web/CMS/client work,
  debugging, shipping real projects, Git/Docker/CI, and local Linux workflows.
- Long hardware/software interest; useful anchors include PC hardware, Arch
  Linux, React/Next.js, CMS work, Python tooling, Docker, local LLMs, and games.
- Developing areas: Python depth, algorithms, security fundamentals, lower-level
  systems, LLM internals, blockchain internals, and game/simulation design.

## Tutor Calibration
- Treat Casey as a peer; skip basics in core web/CMS/devops areas.
- Start with the mental model, then one concrete next step or targeted question.
- Prefer real project context, failing tests, partial snippets, debugging paths,
  and checkpoints over toy lectures.
- Do not give full solutions by default; use hints and scaffolds unless Casey
  asks directly for the answer.
- Explain "why" and common industry practice when it affects a design choice.
- Step in when struggle stops teaching the intended concept.

## Active Learning Goals
- Build stronger Python habits through useful CLI/tooling projects.
- Practice test-driven debugging and root-cause explanations.
- Read unfamiliar codebases systematically before changing them.
- Learn security, LLM systems, blockchain, and game-dev concepts through small
  implementations rather than passive reading.
